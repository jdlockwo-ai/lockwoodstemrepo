ADM Unit 3

Digital Systems and Virtual Integration

Students explore digital twins, virtual manufacturing environments, robot kinematics, simulation, collision prevention, conveyor flow, offline programming, and digital twin analysis.

View Lessons Unit Challenge Back to ADM
Unit Goal

Test before production

This unit focuses on how digital systems help engineers simulate, analyze, troubleshoot, and improve manufacturing processes before physical production begins.

Essential Questions

  • How can a digital twin represent a real manufacturing system?
  • How do virtual environments reduce risk before production?
  • How can simulation improve safety, motion, timing, and layout?
  • How can digital evidence support process improvement decisions?

Learning Targets

  • I can explain the purpose of a digital twin in manufacturing.
  • I can analyze robot motion, kinematics, and workcell layout in a virtual system.
  • I can identify collision risks and process inefficiencies using simulation evidence.
  • I can document digital testing, revisions, and final recommendations.

Major Deliverables

  • Engineering notebook entries
  • Digital twin or virtual workcell evidence
  • Simulation screenshots or motion evidence
  • Collision/risk analysis
  • Process improvement recommendation
  • Factory in Motion reflection
Lesson sequence

Unit 3 lesson clusters

The unit moves from digital twin development to simulation, offline programming, digital twin analysis, and a factory-wide motion challenge.

3.1

Digital Twin Development, Kinematics & Robot Selection

Students learn how digital manufacturing systems are built, represented, and analyzed before physical production begins.

Activity 3.1.1Digital Twin Development

Learn what a digital twin is and how virtual models can represent real manufacturing systems.

Activity 3.1.2Virtual World

Explore how virtual manufacturing environments can test layouts, workflows, and system behavior.

Activity 3.1.3Modeling Kinematics

Investigate how robot motion can be represented and analyzed using kinematics.

Activity 3.1.4The Right Robot for the Job

Compare robot options using task requirements, reach, payload, precision, and workspace.

Project 3.1.5An Incomplete Facility

Analyze a partially developed manufacturing facility and recommend improvements using digital twin and automation concepts.

3.2

Simulation, Motion Control & Offline Programming

Students use digital simulation tools to test robot motion, prevent collisions, control conveyor flow, and program systems virtually.

Activity 3.2.1Simulation Kinematics

Use simulation to analyze robot movement, joint motion, and workcell behavior before physical testing.

Activity 3.2.2Don’t Crash!

Identify collision risks and use digital tools to prevent unsafe or inefficient robot motion.

Activity 3.2.3Conveyor Flow and Motion Control

Study how conveyors, timing, and motion control affect material flow through a manufacturing system.

Activity 3.2.4Offline Programming

Explore how robots and automated systems can be programmed virtually before physical operation.

Activity 3.2.5Sorting Silicon Boules

Apply sorting logic and digital manufacturing concepts to a semiconductor-style production task.

Activity 3.2.6Digital Twin Analysis

Analyze a digital twin to identify problems, evaluate performance, and recommend improvements.

Project 3.2.7Circuit Assembly Challenge

Use virtual integration, motion control, and process planning to solve a circuit assembly manufacturing challenge.

3.3

Factory in Motion Challenge Problem

Students combine digital systems, virtual integration, motion analysis, and workflow improvement in a final unit challenge.

Problem 3.3.1Factory in Motion

Design, simulate, analyze, or improve a virtual manufacturing system that shows how products, robots, conveyors, tools, machines, and workcell layouts interact over time.
